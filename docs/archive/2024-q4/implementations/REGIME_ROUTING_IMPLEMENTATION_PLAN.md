**Expected Impact**:
- 2022 PF: 0.11 → 1.32 (12x improvement)
- 2024 PF: 2.65 → 3.15 (+18% improvement)
- Archetype diversity: 96.5% trap → 40% long_squeeze + 26.7% rejection

**Risk**: LOW (code is functional, just needs config data)

**Timeline**: 1-2 days (config creation + validation)

## Decision Matrix

### Implementation Approach

**Chosen**: Score Multiplication (Option A)

**Rationale**:
- Already implemented in code (line 422: `adjusted_score = score * regime_mult`)
- Simple and transparent
- No risk of score overflow (scores already in [0, 1])
- Fallback to threshold adjustment available if needed

**Alternative**: Threshold Adjustment (Option B)
- More complex (requires ThresholdPolicy integration)
- Better for preserving score interpretability
- Use if score multiplication insufficient

### Weight Aggressiveness

**Chosen**: 0.2x suppression (aggressive)

**Rationale**:
- Sensitivity analysis shows 0.2x is sweet spot (PF 1.32)
- 0.3x too weak (PF 1.05, still marginal)
- 0.1x over-suppression (reduces bear archetype quality)

**Fallback**: Start with 0.3x if validation fails, adjust up/down

### Bear Archetypes to Enable

**Minimum**: S2 (Rejection), S5 (Long Squeeze)
- Proven performers (S2: 58.5% WR, PF 1.4)
- Don't require exotic data (just price/volume/RSI)

**Optional**: S1 (Breakdown), S8 (Volume Fade Chop)
- Add if minimum set insufficient
- S5 currently disabled (needs funding rate data)

**Defer**: S3, S4, S6, S7 (require feature backfill)

## Risk Mitigation

### Risk 1: Regime Misclassification
**Impact**: High (routing won't work if 2022 labeled risk_on)
**Mitigation**:
- Validate regime distribution FIRST (Phase 2.1)
- If wrong, retrain regime classifier before deploying routing
- Use forced regime override for testing: `--force-regime risk_off`

### Risk 2: Bear Archetype Underperformance
**Impact**: High (routing shifts to underperforming patterns)
**Mitigation**:
- Test bear archetypes standalone BEFORE integration (Phase 2.2)
- Set minimum bar: PF >1.0, WR >45%
- Disable underperforming archetypes, keep only S2 if needed

### Risk 3: Over-Suppression
**Impact**: Medium (too few trades in bear markets)
**Mitigation**:
- Monitor trade count (should be 20-30 in 2022, not <10)
- If trade count <15, reduce suppression (0.2 → 0.3)
- Use frontier optimization to find sweet spot

### Risk 4: 2024 Degradation
**Impact**: Medium (production performance drops)
**Mitigation**:
- Safety check MANDATORY before deployment (Phase 2.4)
- Rollback if 2024 PF <2.5
- Adjust risk_on boost weights if needed (1.3 → 1.5)

### Risk 5: Whipsaw During Transitions
**Impact**: Low (frequent regime switches cause thrashing)
**Mitigation**:
- Use regime hysteresis (already in RuntimeContext)
- Monitor Q1 2024 and Q4 2023 transition periods
- Add regime transition cooldown if needed
