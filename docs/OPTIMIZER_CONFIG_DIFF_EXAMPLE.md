### 🔄 Dual-Location Write

Each archetype parameter is written to TWO locations:

**Example: trap_within_trend fusion_threshold**

```json
{
  // Location 1: thresholds subdirectory
  "archetypes.thresholds.trap_within_trend.fusion_threshold": 0.38,

  // Location 2: top-level archetype config
  "archetypes.trap_within_trend.final_fusion_gate": 0.38
}
```

**Why?**
- `ThresholdPolicy._build_base_map()` reads from `archetypes[pattern]` FIRST
- Falls back to `archetypes.thresholds[pattern]` for legacy compatibility
- Dual-write ensures parameters are found regardless of read path

## Code Path Verification

### How ThresholdPolicy Reads Parameters

```python
# engine/archetypes/threshold_policy.py

def _build_base_map(self) -> Dict[str, Dict[str, float]]:
    base_map = {}
    archetypes = self.base.get('archetypes', {})

    for arch_name in ARCHETYPE_NAMES:
        # PR#6A: Try top-level archetype config FIRST (where optimizer writes)
        if arch_name in archetypes and isinstance(archetypes[arch_name], dict):
            if arch_name not in ('use_archetypes', 'thresholds', 'exits'):
                base_map[arch_name] = deepcopy(archetypes[arch_name])
                logger.debug(f"[PR#6A] Loaded {arch_name} from top-level config")
                continue  # ✅ Found in location 2

        # Try descriptive name in thresholds subdirectory (LEGACY)
        if arch_name in self.base_arch_thresholds:
            base_map[arch_name] = deepcopy(self.base_arch_thresholds[arch_name])
            logger.debug(f"[LEGACY] Loaded {arch_name} from thresholds/{arch_name}")
            # ✅ Found in location 1
```

### Result

With dual-location write:
- Parameters in `archetypes[pattern]` are read FIRST (location 2)
- If not found, falls back to `archetypes.thresholds[pattern]` (location 1)
- This ensures optimizer-written params are always used

## Key Takeaways

1. **Safety Rails Always Enforced**
   - `crisis_fuse.enabled` = true (never changed)
   - `max_portfolio_risk_pct` = 0.10 (never changed)
   - VIX/DXY/MOVE thresholds enforced at runtime

2. **Dual-Location Write Works**
   - Parameters in `archetypes[pattern]` (read first)
   - Parameters in `archetypes.thresholds[pattern]` (fallback)
   - ThresholdPolicy finds optimizer-written params

3. **Group-Based Optimization**
   - Only group archetypes enabled during trial
   - Prevents interference between archetype groups
   - Final config aggregates best params from all groups

4. **Metadata Tracking**
   - Optimization timestamp recorded
   - Trial counts and scores saved
   - Safety rails documented in output
   - Clear audit trail for parameter provenance
